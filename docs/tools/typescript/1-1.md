# 语言类型

## 强类型与弱类型

- **强类型与弱类型语言从`类型安全`的维度进行区分**。区分点在于`是否支持隐形类型转化`
> 弱类型：<span class=line1>支持隐式类型转换</span>（松散）。 
>
> 强类型：<span class=line1>不支持隐式类型转换</span>。（严格）

```js
let n = 1;
let m = 'a'

//  √  弱类型：支持 m + n
//  ×  强类型：不支持 m + n
```


## 动态类型与静态类型

- **动态类型与静态类型语言从`类型检查`的维度进行区分**。

> 静态类型：变量使用前需声明数据类型，程序运行过程中<span class="line1">数据类型不允许改变</span>。（严格）
> 
> 动态类型：变量使用前不需要声明数据类型，程序运行过程中<span class="line1">数据类型可以改变</span>。（松散）

```js
let n = 1;

//  x  静态类型：不支持 n = 'a'
//  √  动态类型：支持 n = 'a'
```


##  强类语言的优势
- 提早暴露编码错误
- 编码更准备可靠
- 减少不必要的类型判断



## Javascript弱类动态语言

脚本语言没有编译环节，直接在运行环境中运行。而静态类型语言需要在编译阶段进行类型检查（Javascript没有指定类型）。所以 Javascript 被设计成了`弱类动态语言`。
